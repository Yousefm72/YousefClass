كيفية فتح موجه الأوامر
- اضغط Win ثم اكتب cmd واختر "تشغيل كمسؤول" للحصول على صلاحيات التعديل على الشبكة.

ipconfig — عرض إعدادات شبكة IPv4/IPv6

يعرض عناوين IP وأقنعة الشبكة والبوابة الافتراضية لكل محوّل شبكة.

ipconfig
ipconfig /all            ^ يعرض تفاصيل شاملة (DNS, DHCP, MAC)
ipconfig /release        ^ تحرير (إسقاط) عنوان DHCP
ipconfig /renew          ^ طلب عنوان جديد من DHCP
ipconfig /displaydns     ^ عرض ذاكرة أسماء DNS المحلية
استخدم ipconfig /all لمعرفة ما إذا كان الجهاز يحصل على IP من DHCP واسم خادم DNS.

ping — اختبار الوصول والكمون

يرسل حزم ICMP للتحقق من الوصول إلى مضيف وقياس زمن الاستجابة.

ping 8.8.8.8              ^ اختبار الوصول إلى خادم Google DNS عبر IP
ping www.example.com      ^ اختبار عبر اسم نطاق (يتطلب حل DNS)
ping -n 10 1.1.1.1        ^ إرسال 10 طلبات بدلاً من 4 الافتراضية
ping -l 1024 8.8.4.4      ^ حجم حمولة 1024 بايت لاختبار الاستقرار
إذا نجح ping على IP وفشل على اسم نطاق، فالمشكلة غالبًا في DNS.

tracert — تتبع المسار إلى وجهة

يعرض القفزات (الموجّهات) بينك وبين الوجهة لتحديد مكان الانقطاع أو البطء.

tracert 8.8.8.8
tracert www.example.com
القفزة التي تتوقف عندها الاستجابة قد تشير إلى جهاز يرفض ICMP أو انقطاع في هذا المسار.

nslookup — الاستعلام عن DNS

يستخدم لحل أسماء النطاقات إلى عناوين IP والعكس عبر خوادم DNS.

nslookup www.example.com          ^ الاستعلام باستخدام DNS الافتراضي
nslookup www.example.com 8.8.8.8 ^ الاستعلام باستخدام خادم محدد
nslookup -type=MX example.com     ^ سجلات البريد لنطاق
إذا أعاد nslookup نتيجة بينما فشل المتصفح، فربما توجد مشكلة وكيل Proxy أو جدار ناري.

arp — جدول ربط IP إلى MAC في الشبكة المحلية

يعرض ويدير جدول ARP الذي يربط عناوين IP بعناوين MAC داخل نفس الشبكة المحلية.

arp -a               ^ عرض جدول ARP الحالي
arp -d *            ^ حذف كل السجلات المؤقتة (يتطلب أذونات)
جدول ARP مفيد لكشف التعارضات أو انتحال العناوين داخل الشبكة المحلية.

netstat — المنافذ والاتصالات النشطة

يعرض الاتصالات المفتوحة والمنافذ المستمعة وجداول التوجيه.

netstat -ano         ^ كل الاتصالات مع PID العملية
netstat -an | find "LISTEN"  ^ إظهار المنافذ التي في وضع الاستماع
netstat -r          ^ عرض جدول التوجيه (Route Table)
استخدم PID مع "إدارة المهام" لمعرفة أي برنامج يستخدم منفذًا معيّنًا.

route — إدارة جدول التوجيه

اعرض أو أضف أو احذف مسارات يدوية للشبكات.

route print
route add 10.10.10.0 mask 255.255.255.0 192.168.1.1 metric 10
route delete 10.10.10.0
تعديل المسارات يتطلب تشغيل CMD كمسؤول. استخدم route print قبل وبعد للتأكد.

getmac — عرض عناوين MAC

يعرض عناوين MAC للمحولات المحلية وربما البعيدة (عند توفرها).

getmac /v            ^ عرض مفصّل يتضمن أسماء المحولات

ipconfig /flushdns — مسح ذاكرة DNS المؤقتة

يحل مشاكل التخزين المؤقت لسجلات DNS القديمة.

ipconfig /displaydns
ipconfig /flushdns   ^ مسح الكاش
ipconfig /displaydns ^ التحقق أن الكاش أصبح فارغًا تقريبًا

netsh interface ip show config — إعدادات بروتوكول IP

يعرض تكوينات واجهات الشبكة بالتفصيل، ويمكن عبر netsh أيضًا ضبط IP وDNS.

netsh interface ip show config
netsh interface ipv4 show addresses
rem أمثلة ضبط (تتطلب مسؤول):
rem netsh interface ip set address name="Ethernet" static 192.168.1.50 255.255.255.0 192.168.1.1
rem netsh interface ip set dns name="Ethernet" static 8.8.8.8
نصيحة: عند استكشاف مشاكل الشبكة، اتبع تسلسلًا منطقيًا: تحقق من ipconfig، ثم ping البوابة، ثم DNS عبر nslookup، وبعدها تتبع المسار tracert، ولا تنس فحص المنافذ بـ netstat.
