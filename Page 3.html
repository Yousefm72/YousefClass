البيانات (The Data)

البيانات هي جميع الحقائق أو الأرقام أو الرموز التي يمكن جمعها وتخزينها ومعالجتها باستخدام الحاسوب أو أي وسيلة أخرى. البيانات قد تكون أرقامًا، نصوصًا، صورًا، أصواتًا، أو أي نوع آخر من المعلومات الخام التي لم يتم معالجتها بعد.

أجزاء البيانات

  • البيانات الرقمية: مثل الأرقام (درجات الطلاب، أسعار المنتجات).
  • البيانات النصية: مثل الأسماء، العناوين، الرسائل.
  • البيانات الصورية: مثل الصور الفوتوغرافية، الرسومات.
  • البيانات الصوتية: مثل التسجيلات الصوتية، الموسيقى.
  • البيانات المرئية: مثل مقاطع الفيديو.

تركيب الرسالة (Message Structure)

الرسالة هي وحدة أساسية لنقل البيانات بين المرسل والمستقبل في أنظمة الاتصالات أو الحاسوب. تتكون الرسالة عادة من عدة أجزاء رئيسية تضمن وصول البيانات بشكل صحيح وسليم.

  • المُرسِل (Sender): الجهة أو الجهاز الذي يرسل الرسالة.
  • المستقبِل (Receiver): الجهة أو الجهاز الذي يستقبل الرسالة.
  • المحتوى (Content): البيانات أو المعلومات الفعلية التي يتم إرسالها (مثل نص، صورة، ملف).
  • العنوان (Address): يحدد وجهة الرسالة (مثل عنوان بريد إلكتروني أو عنوان IP).
  • الترويسة (Header): معلومات إضافية عن الرسالة مثل نوعها، تاريخ الإرسال، حجم البيانات.
  • التذييل (Footer): قد يحتوي على معلومات للتحقق من صحة الرسالة أو إنهاءها.

مثال توضيحي:
عند إرسال بريد إلكتروني، يكون المُرسِل هو عنوان البريد الخاص بك، والمستقبِل هو عنوان الشخص الآخر، والمحتوى هو نص الرسالة أو المرفقات، والترويسة تتضمن معلومات مثل عنوان الرسالة وتاريخ الإرسال.

أنواع الرسائل في الشبكات: Unicast و Multicast و Broadcast

عند إرسال رسالة عبر الشبكة، يحدد نوع الإرسال عدد وأسلوب وصولها إلى الأجهزة الهدف. فيما يلي الأنواع الثلاثة الأكثر شيوعًا:

  • Unicast (أحادي الإرسال): رسالة من مُرسِل واحد إلى مُستقبِل واحد محدد.
    مثال عملي: جهازك يطلب صفحة من خادم موقع معيّن؛ الحزم تتجه إلى عنوان IP محدد واحد، وإطار الإيثرنت يحمل عنوان MAC خاص بالمستقبِل.
  • Multicast (متعدد الإرسال إلى مجموعة): رسالة واحدة تُرسل إلى مجموعة أجهزة مشترِكة في عنوان متعدد البث.
    مثال عملي: بث درس مباشر داخل شبكة المدرسة إلى الصفوف المشترِكة على عنوان متعدد البث مثل 239.1.1.1؛ الأجهزة المشترِكة فقط هي التي تستقبل.
  • Broadcast (بث عام محلي): رسالة تُرسل إلى جميع الأجهزة داخل نفس الشبكة المحلية (القطاع الفرعي).
    مثال عملي: استعلام ARP "من صاحب IP 192.168.1.20؟" يُرسل إلى عنوان MAC العام ff:ff:ff:ff:ff:ff بحيث تستقبله كل الأجهزة في الشبكة المحلية.
ملاحظات سريعة
  • البث العام Broadcast لا يتجاوز حدود الشبكة المحلية عادةً، بينما Unicast و Multicast يمكن توجيههما بين الشبكات.
  • نطاق عناوين IP متعدد البث هو 224.0.0.0 إلى 239.255.255.255، ويستخدم بروتوكولات مثل IGMP للاشتراك.

أجزاء حزمة البيانات وعلاقتها بطبقات OSI

عند إرسال البيانات عبر الشبكة، تمر عبر طبقات نموذج OSI المختلفة، وتتخذ أشكالاً مختلفة في كل طبقة. كل طبقة تضيف معلومات تحكم خاصة بها تسمى "ترويسة" (Header).

البيانات (Data) - طبقة التطبيق (Layer 7)

الوصف: البيانات الأصلية التي يريد المستخدم إرسالها (نص، صورة، ملف، إلخ).
مثال: "مرحباً بك في الشبكة" أو محتوى صفحة ويب.

Data: "Hello World" (11 bytes)

القطعة (Segment) - طبقة النقل (Layer 4)

الوصف: البيانات + ترويسة TCP أو UDP تحتوي على معلومات المنفذ ورقم التسلسل.
مثال: TCP Segment مع منفذ المصدر والهدف.

TCP Header (20 bytes): Source Port, Dest Port, Sequence, Checksum
Data: "Hello World" (11 bytes)
Total Segment: 31 bytes

الحزمة (Packet) - طبقة الشبكة (Layer 3)

الوصف: القطعة + ترويسة IP تحتوي على عناوين IP المصدر والهدف.
مثال: IP Packet مع عناوين IPv4.

IP Header (20 bytes): Source IP, Dest IP, TTL, Protocol
TCP Segment: 31 bytes
Total Packet: 51 bytes

الإطار (Frame) - طبقة ربط البيانات (Layer 2)

الوصف: الحزمة + ترويسة Ethernet تحتوي على عناوين MAC المصدر والهدف.
مثال: Ethernet Frame مع عناوين MAC.

Ethernet Header (14 bytes): Dest MAC, Source MAC, Type
IP Packet: 51 bytes
FCS (4 bytes): Error Detection
Total Frame: 69 bytes

البت (Bit) - الطبقة الفيزيائية (Layer 1)

الوصف: الإطار مُحول إلى إشارات كهربائية أو ضوئية (0 و 1).
مثال: إشارات كهربائية على كابل Ethernet أو إشارات ضوئية في الألياف البصرية.

Bits: 01001000 01100101 01101100 01101100 01101111...
Total: 69 bytes × 8 = 552 bits
ملخص العملية
  1. طبقة التطبيق: البيانات الأصلية (11 بايت)
  2. طبقة النقل: + ترويسة TCP = القطعة (31 بايت)
  3. طبقة الشبكة: + ترويسة IP = الحزمة (51 بايت)
  4. طبقة ربط البيانات: + ترويسة Ethernet + FCS = الإطار (69 بايت)
  5. الطبقة الفيزيائية: تحويل إلى بتات (552 بت)
