نموذج OSI (Open Systems Interconnection)

نموذج OSI هو إطار عمل نظري يوضح كيفية عمل الشبكات وتبادل البيانات بين الأجهزة. تم تطويره من قبل المنظمة الدولية للتوحيد القياسي (ISO) في عام 1984، ويقسم عملية الاتصال الشبكي إلى سبع طبقات منطقية، كل طبقة تتعامل مع جانب محدد من عملية الاتصال.

يساعد هذا النموذج المطورين والمهندسين على فهم كيفية عمل الشبكات، وتصميم بروتوكولات جديدة، وحل المشاكل الشبكية بطريقة منظمة ومنهجية.

الطبقات السبع لنموذج OSI

يتم ترتيب الطبقات من الأعلى (طبقة التطبيق) إلى الأسفل (طبقة الفيزيائية). كل طبقة تقدم خدمات للطبقة التي فوقها، وتتلقى خدمات من الطبقة التي تحتها.

7
طبقة التطبيق (Application Layer)
Application Layer

هذه هي الطبقة التي يتعامل معها المستخدمون مباشرة. تقدم خدمات الشبكة للتطبيقات مثل متصفحات الويب، برامج البريد الإلكتروني، وبرامج نقل الملفات.

البروتوكولات:

HTTP, HTTPS, FTP, SMTP, POP3, IMAP, DNS, DHCP

وحدة البيانات: Message

المسؤوليات:

  • تقديم واجهة المستخدم للشبكة
  • إدارة جلسات الاتصال
  • تحديد متطلبات التطبيق
  • التعامل مع الأخطاء والتطبيقات
6
طبقة العرض (Presentation Layer)
Presentation Layer

تتعامل هذه الطبقة مع تنسيق البيانات وعرضها، وضمان أن البيانات المرسلة من طبقة التطبيق يمكن قراءتها من قبل طبقة التطبيق في الجهاز المستقبل.

البروتوكولات:

SSL/TLS, JPEG, PNG, MPEG, ASCII, Unicode

وحدة البيانات: Message

المسؤوليات:

  • تشفير وفك تشفير البيانات
  • ضغط وفك ضغط البيانات
  • تحويل تنسيق البيانات
  • إدارة الترميز (Encoding)
5
طبقة الجلسة (Session Layer)
Session Layer

تدير هذه الطبقة جلسات الاتصال بين التطبيقات، وتتحكم في فتح وإغلاق الاتصالات، ومزامنة البيانات بين الأجهزة.

البروتوكولات:

NetBIOS, RPC, SQL, NFS

وحدة البيانات: Message

المسؤوليات:

  • إنشاء وإدارة الجلسات
  • مزامنة البيانات
  • إدارة نقاط الفحص (Checkpoints)
  • استئناف الجلسات المتوقفة
4
طبقة النقل (Transport Layer)
Transport Layer

تضمن هذه الطبقة نقل البيانات بشكل موثوق ودقيق بين نقاط النهاية، وتتحكم في تدفق البيانات وإدارة الأخطاء.

البروتوكولات:

TCP, UDP, SCTP, DCCP

وحدة البيانات: Segment (TCP) / Datagram (UDP)

المسؤوليات:

  • ضمان موثوقية نقل البيانات
  • إدارة تدفق البيانات
  • اكتشاف وإصلاح الأخطاء
  • إعادة ترتيب الحزم الواردة
3
طبقة الشبكة (Network Layer)
Network Layer

تتعامل هذه الطبقة مع توجيه البيانات عبر الشبكة، وتحديد أفضل مسار للبيانات للوصول إلى وجهتها.

البروتوكولات:

IP, ICMP, OSPF, BGP, RIP, ARP

وحدة البيانات: Packet

المسؤوليات:

  • توجيه الحزم عبر الشبكة
  • إدارة عناوين IP
  • تحديد أفضل المسارات
  • إدارة الازدحام الشبكي
2
طبقة ربط البيانات (Data Link Layer)
Data Link Layer

تتعامل هذه الطبقة مع نقل البيانات بين الأجهزة المتصلة مباشرة، وتتحكم في الوصول إلى الوسائط المشتركة.

البروتوكولات:

Ethernet, Wi-Fi, PPP, Frame Relay, ATM

وحدة البيانات: Frame

المسؤوليات:

  • إدارة الوصول إلى الوسائط
  • اكتشاف وإصلاح الأخطاء
  • إدارة عناوين MAC
  • ضمان نقل البيانات الموثوق
1
الطبقة الفيزيائية (Physical Layer)
Physical Layer

هذه هي الطبقة الأساسية التي تتعامل مع الوسائط الفيزيائية (الكابلات، الإشارات اللاسلكية) ونقل البتات (0 و 1).

البروتوكولات:

Ethernet, Wi-Fi, Bluetooth, USB, HDMI

وحدة البيانات: Bit

المسؤوليات:

  • نقل البتات عبر الوسائط
  • تحديد خصائص الوسائط
  • إدارة الإشارات الكهربائية
  • تحديد التوصيلات والموصلات

تدفق البيانات عبر طبقات OSI

عندما يتم إرسال بيانات من جهاز إلى آخر، تمر البيانات عبر جميع الطبقات السبع في الجهاز المرسل، ثم عبر الوسائط، وأخيراً عبر جميع الطبقات في الجهاز المستقبل.

مثال على تدفق البيانات:

عند إرسال رسالة بريد إلكتروني:

  1. طبقة التطبيق: برنامج البريد الإلكتروني يجهز الرسالة
  2. طبقة العرض: تشفير الرسالة وتنسيقها
  3. طبقة الجلسة: إنشاء جلسة اتصال مع خادم البريد
  4. طبقة النقل: تقسيم الرسالة إلى أجزاء وإضافة أرقام التسلسل
  5. طبقة الشبكة: إضافة عناوين IP للمصدر والوجهة
  6. طبقة ربط البيانات: إضافة عناوين MAC وإرسال الإطارات
  7. الطبقة الفيزيائية: تحويل البيانات إلى إشارات كهربائية

مقارنة نموذج OSI مع TCP/IP

نموذج TCP/IP هو نموذج عملي يستخدم في الإنترنت الفعلي، بينما نموذج OSI هو نموذج نظري. إليك مقارنة بينهما:

طبقة OSI طبقة TCP/IP البروتوكولات

6. العرض 4. التطبيق SSL/TLS, JPEG, MPEG
5. الجلسة 4. التطبيق NetBIOS, RPC
4. النقل 3. النقل TCP, UDP
3. الشبكة 2. الإنترنت IP, ICMP, ARP
2. ربط البيانات 1. الوصول للشبكة Ethernet, Wi-Fi
1. الفيزيائية 1. الوصول للشبكة الكابلات, الإشارات

أهمية نموذج OSI

نموذج OSI مهم لعدة أسباب:

  • التعليم: يساعد في فهم كيفية عمل الشبكات
  • التطوير: يوجه تطوير بروتوكولات جديدة
  • استكشاف الأخطاء: يساعد في تحديد مكان المشاكل الشبكية
  • التوافق: يضمن توافق المنتجات من مختلف الشركات
  • التوثيق: يوفر لغة مشتركة للمهندسين

فهم نموذج OSI أساسي لأي شخص يعمل في مجال الشبكات، سواء كان مهندس شبكات، مطور برمجيات، أو مستخدم عادي يريد فهم كيفية عمل الإنترنت.
