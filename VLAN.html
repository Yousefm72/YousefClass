VLAN - الشبكات الافتراضية المحلية

VLAN (Virtual Local Area Network) هي تقنية تسمح بتقسيم الشبكة الفيزيائية إلى عدة شبكات منطقية منفصلة. هذا التقسيم يحسن الأمان والأداء وإدارة الشبكة بشكل كبير.

ما هو VLAN؟

تعريف VLAN

VLAN هي شبكة افتراضية تنشأ داخل موزع أو عدة موزعات متصلة. تسمح بتجميع الأجهزة منطقياً بغض النظر عن موقعها الفيزيائي في الشبكة.

مميزات استخدام VLAN

المميزات الرئيسية

  • تحسين الأمان: عزل حركة البيانات بين الأقسام المختلفة
  • تقليل حركة البث: تقليل عدد الرسائل المرسلة لجميع الأجهزة
  • سهولة الإدارة: إمكانية إعادة تنظيم الشبكة بدون تغيير الكابلات
  • تحسين الأداء: تقليل التصادمات وزيادة الكفاءة
  • المرونة: إمكانية نقل الأجهزة بين الأقسام بسهولة

أنواع VLAN

1. VLAN الافتراضي (Default VLAN)

VLAN 1

جميع المنافذ في الموزع تنتمي إلى VLAN 1 افتراضياً. هذا الـ VLAN يحتوي على جميع الأجهزة المتصلة قبل تكوين VLANs أخرى.

2. VLAN المستخدم (User VLAN)

VLANs مخصصة

  • VLAN 10: قسم المبيعات
  • VLAN 20: قسم المحاسبة
  • VLAN 30: قسم الموارد البشرية
  • VLAN 40: قسم تقنية المعلومات

3. VLAN الإدارة (Management VLAN)

VLAN للإدارة

VLAN مخصص لإدارة أجهزة الشبكة مثل الموزعات والموجهات. عادة ما يكون VLAN 99 أو أي رقم آخر غير 1.

تكوين VLAN

إنشاء VLAN جديد

Switch# configure terminal
Switch(config)# vlan 10
Switch(config-vlan)# name Sales
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name Accounting
Switch(config-vlan)# exit

تعيين منفذ إلى VLAN

Switch(config)# interface fastethernet 0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
Switch(config-if)# exit

Switch(config)# interface range fastethernet 0/2-5
Switch(config-if-range)# switchport mode access
Switch(config-if-range)# switchport access vlan 20
Switch(config-if-range)# exit

TRUNK - الجذع

تعريف TRUNK

TRUNK هو رابط عالي السرعة يحمل حركة بيانات عدة VLANs بين الموزعات أو بين الموزع والموجه. يستخدم تقنية التوسيم (Tagging) لتمييز البيانات التي تنتمي لكل VLAN.

لماذا نحتاج TRUNK؟

المشكلة بدون TRUNK

  • الحاجة لعدة كابلات: كابل منفصل لكل VLAN
  • إهدار المنافذ: استهلاك منافذ كثيرة في الموزعات
  • التكلفة العالية: تكلفة إضافية للكابلات والمنافذ
  • صعوبة الإدارة: تعقيد في إدارة الكابلات

الحل مع TRUNK

  • كابل واحد: كابل واحد يحمل جميع VLANs
  • توفير المنافذ: استخدام منفذ واحد بدلاً من عدة منافذ
  • تقليل التكلفة: توفير في الكابلات والمنافذ
  • سهولة الإدارة: إدارة مبسطة للروابط

تكوين TRUNK

Switch# configure terminal
Switch(config)# interface fastethernet 0/24
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk encapsulation dot1q
Switch(config-if)# switchport trunk allowed vlan 10,20,30
Switch(config-if)# exit

أوامر فحص TRUNK

Switch# show interfaces trunk
Switch# show interfaces fastethernet 0/24 switchport
Switch# show vlan brief

أمثلة عملية

مثال 1: شركة صغيرة

السيناريو

شركة لديها 3 أقسام: المبيعات (5 أجهزة)، المحاسبة (3 أجهزة)، تقنية المعلومات (2 أجهزة). جميع الأجهزة متصلة بموزع واحد.

تكوين VLANs:
Switch(config)# vlan 10
Switch(config-vlan)# name Sales
Switch(config-vlan)# exit

Switch(config)# vlan 20
Switch(config-vlan)# name Accounting
Switch(config-vlan)# exit

Switch(config)# vlan 30
Switch(config-vlan)# name IT
Switch(config-vlan)# exit

تعيين المنافذ:
Switch(config)# interface range fa0/1-5
Switch(config-if-range)# switchport access vlan 10

Switch(config)# interface range fa0/6-8
Switch(config-if-range)# switchport access vlan 20

Switch(config)# interface range fa0/9-10
Switch(config-if-range)# switchport access vlan 30

مثال 2: شبكة متعددة الموزعات

السيناريو

شركة لديها موزعين متصلين ببعضهما. الموزع الأول يحتوي على قسم المبيعات، والموزع الثاني يحتوي على قسم المحاسبة. نحتاج لربط القسمين عبر TRUNK.

الموزع الأول (Switch1):
Switch1(config)# vlan 10
Switch1(config-vlan)# name Sales
Switch1(config-vlan)# exit

Switch1(config)# interface fa0/24
Switch1(config-if)# switchport mode trunk
Switch1(config-if)# switchport trunk allowed vlan 10,20

الموزع الثاني (Switch2):
Switch2(config)# vlan 20
Switch2(config-vlan)# name Accounting
Switch2(config-vlan)# exit

Switch2(config)# interface fa0/24
Switch2(config-if)# switchport mode trunk
Switch2(config-if)# switchport trunk allowed vlan 10,20

أوامر الفحص والتحقق

عرض جميع VLANs:
Switch# show vlan brief

عرض حالة TRUNK:
Switch# show interfaces trunk

عرض تفاصيل منفذ:
Switch# show interfaces fa0/24 switchport

عرض MAC addresses لكل VLAN:
Switch# show mac address-table vlan 10
